Issue #9882: Source Map Generation

Debug production CSS with full source references in DevTools

Problem

No easemotion.min.css.map is generated during npm run build.

Before: Only easemotion.min.css exists → DevTools shows line 1, column N with no source trace.
After: easemotion.min.css + easemotion.min.css.map → DevTools maps each rule to its original source file.

How to Verify

  1. 1
    Build
    npm run build → generates easemotion.min.css and easemotion.min.css.map
  2. 2
    Check files exist
    ls -la easemotion.min.css*
  3. 3
    Open in DevTools
    Inspect any element → "Styles" panel shows source file names instead of just "easemotion.min.css:1"
  4. 4
    Validate
    node scripts/validate-bundle.mjs checks source map integrity

Affected Files

build-minified-css.mjs

import { minify } from "csso";
const result = minify(css, {
  filename: "easemotion.css",
  sourceMap: true,
});
// Write .css.map alongside .css

validate-bundle.mjs

// Validate source map
const map = JSON.parse(rebuiltMap);
if (map.version !== 3) throw ...
if (!map.sources.length) throw ...
if (!map.mappings.length) throw ...

validate-pack.mjs

"easemotion.min.css.map" // added to required[]
