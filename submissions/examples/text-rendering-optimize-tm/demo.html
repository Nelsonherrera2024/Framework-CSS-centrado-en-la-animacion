text-rendering

Text Rendering Optimization

The text-rendering property gives browsers a hint for optimizing text drawing performance and quality tradeoffs. Essential for high-performance web apps and precise typography.

Rendering Mode Comparison
Each text-rendering value tells the browser to prioritize different aspects: speed, legibility, or geometric precision.
Note: text-rendering is a hint, not a guarantee. Browsers may ignore it or apply it differently. The effects are most noticeable at large font sizes and in SVG contexts.
text-rendering: auto — Browser's default heuristic

The browser balances rendering speed and visual quality based on font size, language, and device pixel ratio. For most UI text, this is the optimal setting. Auto allows the browser to make intelligent decisions about when to apply kerning, ligatures, and hinting.

text-rendering: optimizeSpeed — Prioritize render speed

WAVY HEADLINES

The browser skips kerning and ligature rendering, drawing text as quickly as possible. Best for large amounts of text where visual refinement is less important than smooth scrolling performance. Common in real-time data displays and financial tickers.

text-rendering: optimizeLegibility — Prioritize readability

The browser enables kerning and ligatures for better readability, especially at small text sizes. Essential for body text, multilingual content, and design systems where letterform quality matters. This is the recommended setting for most text-heavy web applications.

text-rendering: geometricPrecision — Exact geometry

The browser renders text with geometric precision, scaling letterforms without rounding to pixel boundaries. Critical for SVG text, responsive typography, and designs where exact proportions are required. Note: this may impact rendering performance on some devices.
