CSS Line Clamp — Text Truncation

Fixes #15571 — self-contained example under submissions/examples/line-clamp-text/. Pure CSS multi-line truncation. Zero JavaScript for the clamp itself.

Fixed Line-Count Classes

.ease-clamp-1

This paragraph will be truncated after exactly one line no matter how much content is written inside it. The ellipsis appears at the end.

.ease-clamp-2

This paragraph will be truncated after exactly two lines. It gives a bit more context than a single line while still keeping the card height predictable and consistent across a grid layout.

.ease-clamp-3

Three lines of text are commonly used in blog post cards and article previews. This gives enough content to understand the topic without overwhelming the reader. The rest is hidden with an ellipsis at the end of line three, keeping layout tight and scannable.

.ease-clamp-4

Four lines work well for product descriptions or long summary text where a bit more context is beneficial. The card height remains uniform because all cards in the grid clamp at the same number of lines, regardless of the actual text length inside each card.

.ease-clamp-5

Five lines are sometimes used for longer previews like news articles, email summaries, or support ticket descriptions. Even with five lines, the layout stays predictable and the grid remains aligned. Any content beyond five lines is hidden by overflow:hidden and the ellipsis character is displayed to signal truncation to the reader.

.ease-clamp-6

Six lines provide a generous preview for content-heavy components like README excerpts, changelog entries, or comment threads. The height is still capped and predictable. Combined with a "Read more" link, this pattern avoids both truncating too aggressively and exposing walls of text in a UI component designed for scanning rather than reading.

Dynamic — CSS Custom Property

3

This paragraph responds to the slider above. Set --ease-clamp-lines on any element that has the .ease-clamp class and the line count updates instantly. No JavaScript animation — just a CSS custom property being changed. You can set this on a parent element to control multiple children, or set it inline as shown here for per-element control. This makes the line clamp utility flexible enough for any design system or component library without hardcoding any specific line count.

No JavaScript — uses hidden checkbox + :checked selector.

EaseMotion CSS is a lightweight, animation-first CSS framework built for the modern web. It provides utility classes for fade, slide, zoom, bounce, float, typewriter, and skeleton effects — all with zero JavaScript, full dark mode support, and prefers-reduced-motion accessibility built in. Every component follows the ease-* naming convention for predictable, collision-free class names. Quick Reference

<!-- Fixed classes -->
<p class="ease-clamp-1">One line</p>
<p class="ease-clamp-2">Two lines</p>
<p class="ease-clamp-3">Three lines</p>

<!-- CSS custom property (any number) -->
<p class="ease-clamp" style="--ease-clamp-lines: 4">Four lines</p>

<!-- Read more toggle (no JS) -->
<div class="ease-clamp-expand">
  <input type="checkbox" id="expand-1" class="ease-clamp-toggle" />
  <p class="ease-clamp-3 ease-clamp-content">Long text...</p>
  <label for="expand-1" class="ease-clamp-btn"></label>
</div>
Browser support: -webkit-line-clamp is supported in Chrome 14+, Firefox 68+, Safari 5+, and Edge 17+ — effectively all modern browsers. The -webkit- prefix is still required even in non-WebKit browsers because the unprefixed line-clamp property is part of the CSS Overflow Level 4 spec and not yet widely available.

The three required properties must always appear together: overflow: hidden, display: -webkit-box, and -webkit-box-orient: vertical.
