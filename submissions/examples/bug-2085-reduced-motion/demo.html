Bug Fix #2085: prefers-reduced-motion in Scroll Progress Bar

The .ease-scroll-progress component now respects the prefers-reduced-motion: reduce accessibility setting.

1. Normal Motion (default)

Scroll the box below to see the animated progress bar. The bar fills from left to right as you scroll.

Scroll inside this box...

Keep scrolling to see the progress

2. Reduced Motion

When the browser has prefers-reduced-motion: reduce set (or via OS accessibility settings), the progress bar is static — no animation runs, WCAG 2.1 SC 2.3.3 compliant.

3. Root Timeline Variant

.ease-scroll-progress-root also inherits the reduced-motion behaviour automatically.
