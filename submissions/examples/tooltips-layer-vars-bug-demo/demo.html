Bug: tooltip vars in wrong layer

:root custom properties are defined inside @layer easemotion-components, giving them lower cascade priority than unlayered CSS. They should be in the base layer.

Hover or focus me

Source

@layer easemotion-components {
  :root {
    --ease-tooltip-bg: var(--ease-color-neutral-900, #1f2937);
    /* ...more vars... */
  }
}

Fix

/* Move to variables.css (easemotion-base layer) */
:root {
  --ease-tooltip-bg: var(--ease-color-neutral-900, #1f2937);
}
