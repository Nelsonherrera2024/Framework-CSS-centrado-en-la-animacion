@layer order declaration — easemotion/all.css

Without an explicit @layer order, browsers assign cascade priority based on the order layers are first encountered — which depends on import order and can differ across bundlers/browsers.

With the declaration below at the top of all.css, the layer order is always: base → components → utilities, regardless of how the file is consumed. 

@layer easemotion-base, easemotion-components, easemotion-utilities;
