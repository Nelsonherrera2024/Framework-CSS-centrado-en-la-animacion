Scroll Entry + Exit Animations

Cards that animate in when they enter the viewport and animate out when they leave — using animation-timeline: view().

Scroll down to see each card fade + slide in as it enters, and fade + slide out as it exits. All powered by CSS scroll-driven animations with animation-range.

Scroll-Driven Animations

CSS scroll-driven animations link animation progress to scroll position rather than time. The view() timeline tracks an element's position relative to the viewport.

Entry Range

animation-range: entry runs the animation from the moment the element starts entering the viewport until it's fully visible.

Exit Range

animation-range: exit runs the animation as the element leaves the viewport — perfect for exit effects.

Combined Entry + Exit

By combining entry and exit animations on the same element, you get a seamless transition — fade in on arrival, fade out on departure.

animation-range: entry 0% exit 100%

This shorthand sets the animation to play across the entire visible range — from first pixel entering to last pixel leaving.

Cross-browser Support

Scroll-driven animations are supported in Chrome 115+. For other browsers, the cards remain visible as a static fallback.

Each card uses animation-timeline: view() with animation-range: entry 0% exit 100%. The @keyframes define both the entry state (opacity 0, translateY) and exit state.
