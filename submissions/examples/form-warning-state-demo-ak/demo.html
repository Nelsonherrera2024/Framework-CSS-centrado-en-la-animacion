Form Warning State Bug

Form warning states use error colors instead of dedicated warning tokens.

Issue #5875

The Problem

When a form field enters a warning state (e.g. password nearly expired, input format ambiguous), the styling incorrectly uses --ease-color-error instead of --ease-color-warning tokens.

Before vs After

Broken — Uses Error Color

Warning Shown as Error

Warning: Password expires in 3 days

Warning state uses --ease-color-error — user may think something is wrong.

Fixed — Uses Warning Token

Proper Warning Styling

Warning: Password expires in 3 days

Warning state uses --ease-color-warning-500 — clear distinction from errors.
