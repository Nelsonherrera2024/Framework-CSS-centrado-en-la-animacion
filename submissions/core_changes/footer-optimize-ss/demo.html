Optimization Strategy

1. Extract SVG icons to a separate file -4.5 KB

The 7 inline SVG data URIs take ~5.5 KB. Move them to an footer-icons.css or use a sprite sheet. Alternatively, use shorter inline SVGs or replace with Unicode/emoji fallbacks.

2. Consolidate light mode vars -1.0 KB

Instead of duplicating 15+ selectors inside @media (prefers-color-scheme: light), define a single .ease-footer rule that uses var(--ease-footer-bg, ...) with light-mode fallbacks via [data-theme="light"].

/* Instead of @media prefers-color-scheme:light with 15 selectors */
.ease-footer { background: var(--ease-footer-bg); }
[data-theme="light"] .ease-footer,
.ease-footer:has([data-theme="light"]) { background: var(--ease-footer-bg-light); }

3. Remove duplicate `transition` declarations -0.5 KB

Many elements declare transition: var(--ease-footer-transition) individually. Apply it at the parent level and use transition: inherit on children.

4. Reduce animated orbs code -0.5 KB

The ::before/::after orbs share most properties. Use a single declaration with a variable for differences.

5. Combine responsive breakpoints -0.5 KB

The 480px and 768px breakpoints can be merged into a single max-width: 768px block with conditional overrides.

6. Use CSS shorthand & remove !important -0.3 KB

Replace margin-left: auto; margin-right: auto with margin-inline: auto. Remove !important in reduced-motion block — increase specificity instead.
