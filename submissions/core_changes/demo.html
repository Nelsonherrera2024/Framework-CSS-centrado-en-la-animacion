.ease-card-info Dark Mode Fix

Bug: color: var(--ease-color-primary-dark, #4b44cc) has no dark-mode override — unreadable on dark backgrounds

Default Card

Default card text — no color override, uses body text color.

Works fine in both modes

Info Card Bug

Info card content — hard to read in dark mode due to #4b44cc on dark background.

Uses --ease-color-primary-dark without dark-mode override

Info Card Fixed

Info card content — lightens in dark mode for proper contrast.

Fixed via @media (prefers-color-scheme: dark) or .ease-dark override

The Fix

Add a dark-mode override for .ease-card-info in the card component stylesheet:

/* Dark mode override for .ease-card-info */
@media (prefers-color-scheme: dark) {
  .ease-card-info,
  .ease-dark .ease-card-info {
    color: var(--ease-color-primary-light, #a09af8);
  }
}
