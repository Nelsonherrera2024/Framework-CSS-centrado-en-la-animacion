CSS Quality Checks

CI pipeline configuration for automated CSS validation, bundle size budgeting, naming convention enforcement, and variable reference validation.

Pipeline Checks

  • ✓ Stylelint — Syntax validation on all CSS files enforced
  • ✓ Bundle Size Budget — Raw ≤ 180 KB, GZip ≤ 32 KB enforced
  • ✓ CSS Variable Validation — All --ease-* refs must exist in core/variables.css enforced
  • ✓ Naming Convention — All classes must use ease- prefix warning
  • ✓ Nesting Depth — Warn if CSS nesting exceeds 3 levels warning

Workflow: css-quality.yml

name: CSS Quality Checks
on:
  push: { branches: [main] }
  pull_request: { branches: [main] }

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20 }
      - run: npm ci
      - run: npx stylelint "**/*.css" --config .stylelintrc.json

  bundle-size:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: |
          FILE="easemotion.min.css"
          RAW_LIMIT=180000; GZIP_LIMIT=32000
          SIZE=$(stat -c%s "$FILE")
          GZIP=$(gzip -c "$FILE" | wc -c)
          [ "$SIZE" -gt "$RAW_LIMIT" ] && exit 1
          [ "$GZIP" -gt "$GZIP_LIMIT" ] && exit 1

  validate-vars:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: |
          grep -oP '--ease-[\w-]+(?=\s*:)' core/variables.css \
            > /tmp/defined.txt
          grep -roP '--ease-[\w-]+' core/ components/ \
            | grep -v core/variables.css \
            | grep -oP '--ease-[\w-]+' | sort -u > /tmp/used.txt
          comm -23 /tmp/used.txt /tmp/defined.txt \
            | awk '{print "Undefined: "$1}'
          [ -s /tmp/missing.txt ] && exit 1 || echo "OK"

Local Usage

Run the same checks locally:

bash submissions/core_changes-issue-12704/css-quality.sh

Copy the workflow to enable CI:

cp submissions/core_changes-issue-12704/css-quality.yml .github/workflows/
