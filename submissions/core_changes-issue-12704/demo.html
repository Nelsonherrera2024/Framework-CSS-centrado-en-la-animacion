CSS Quality CI Pipeline

Automated CSS validation and bundle size checks for EaseMotion CSS. Five independent checks run on every push and pull request to keep the framework healthy.

Checks Overview

1
CSS Lint Stylelint
Runs stylelint on all CSS files using the project's .stylelintrc.json config. Catches syntax errors, duplicate properties, empty blocks, and invalid values.
npx stylelint "**/*.css" --config .stylelintrc.json --allow-empty-input
2
Bundle Size Budget
Ensures easemotion.min.css stays under 180 KB raw and 30 KB gzipped. Prevents bundle bloat from unchecked additions.
SIZE=$(stat -c%s easemotion.min.css) # Linux GZ_SIZE=$(gzip -c easemotion.min.css | wc -c | tr -d ' ')
3
Custom Property Validation Reference
Scans all core/ and components/ CSS files and ensures every --ease-* custom property referenced is defined in core/variables.css.
DEFINED=$(grep -oP '--ease-[\w-]+(?=\s*:)' core/variables.css | sort -u) USED=$(grep -roP '--ease-[\w-]+' core/ components/ | sort -u) # Fail if any USED variable is not in DEFINED
4
Naming Convention ease- prefix
Ensures all CSS classes in core/, components/, and submissions/ use the ease- prefix. Pseudo-classes and keyframe selectors are excluded.
# Scans for .class selectors and verifies they start with ease-
5
Nesting Depth Warning
Warns when CSS nesting exceeds 3 levels — encourages maintainable, flat selectors without deep specificity chains.
# Warns if nesting depth > 3 during brace counting

Workflow

name: CSS Quality Checks
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - name: Stylelint
        run: npx stylelint "**/*.css" --config .stylelintrc.json --allow-empty-input

  bundle-size:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Check bundle size
        run: |
          FILE="easemotion.min.css"
          RAW_LIMIT=180000
          if [ -f "$FILE" ]; then
            SIZE=$(stat -c%s "$FILE")
            if [ "$SIZE" -gt "$RAW_LIMIT" ]; then
              echo "❌ $FILE is $SIZE bytes (limit: $RAW_LIMIT)"
              exit 1
            fi
          fi
      - name: Check gzipped size
        run: |
          FILE="easemotion.min.css"
          GZIP_LIMIT=30720
          if [ -f "$FILE" ]; then
            GZ_SIZE=$(gzip -c "$FILE" | wc -c | tr -d ' ')
            if [ "$GZ_SIZE" -gt "$GZIP_LIMIT" ]; then
              echo "❌ $FILE gzipped is $GZ_SIZE bytes (limit: $GZIP_LIMIT)"
              exit 1
            fi
          fi

  validate-vars:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Check custom property references
        run: |
          DEFINED=$(grep -oP '--ease-[\w-]+(?=\s*:)' core/variables.css | sort -u)
          USED=$(find core components -name '*.css' ! -name 'variables.css' | \
            xargs grep -roP '--ease-[\w-]+' | grep -oP '--ease-[\w-]+' | sort -u)
          MISSING=()
          for var in $USED; do
            if ! echo "$DEFINED" | grep -qF "$var"; then
              MISSING+=("$var")
            fi
          done
          if [ ${#MISSING[@]} -gt 0 ]; then
            echo "❌ Undefined CSS variables:"; printf '  %s\n' "${MISSING[@]}"; exit 1
          fi

  naming-convention:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Check ease- prefix
        run: |
          CSS_FILES=$(find core components submissions -name '*.css')
          for f in $CSS_FILES; do
            grep -oP '\.[a-zA-Z][\w-]*(?=[\s,{])' "$f" | while IFS= read -r cls; do
              cls_name="${cls#.}"
              case "$cls_name" in
                ease-*|active|hover|focus|from|to) ;;
                *) echo "⚠️ $f: '$cls_name' lacks ease- prefix"; exit 1 ;;
              esac
            done
          done

  nesting-depth:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Check nesting depth
        run: |
          for f in $(find core components submissions -name '*.css'); do
            DEPTH=0
            while IFS= read -r line; do
              case "$line" in *{*) DEPTH=$((DEPTH + 1)) ;; *}*) DEPTH=$((DEPTH - 1)) ;; esac
              if [ "$DEPTH" -gt 3 ] && echo "$line" | grep -qP '^\s+\S'; then
                echo "⚠️ $f exceeds 3 nesting levels (depth: $DEPTH)"
              fi
            done < "$f"
          done

Local Usage

Run all checks locally with the included shell script:

chmod +x submissions/core_changes-issue-12704/css-quality.sh
./submissions/core_changes-issue-12704/css-quality.sh
